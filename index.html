About the Workflow

What does this
pipeline do?

This Make.com scenario is built to solve a common research pain: staying on top of a fast-moving literature without spending hours manually searching, triaging, and logging papers.

On each run, it fetches fresh items from an RSS feed (PubMed RSS by default), checks your Google Sheets database to prevent duplicates, then sends the abstract/summary to an LLM to extract a clean JSON payload.

That JSON is sanitized, parsed, and mapped into columns so you get a continuously growing, filterable research database for your own project — whether you’re tracking a biological pathway, a clinical area, a method, or an interdisciplinary topic.

Everything is modular: swap the RSS source, change the extraction fields, and remap the sheet — typically in < 30 minutes.

Adaptation Guide
Key things to change for your project
Module · RSS Reader
RSS source & throughput

Replace the RSS URL with the feed you want to monitor. Adjust the max results per run to control workload and cost.

url: "https://.../rss/search/YOUR_QUERY/?limit=50"
maxResults: "25"
Module · Duplicate Check (Sheets filter)
De-duplication key

Choose the column you want to match on (paper URL, DOI, or GUID). The default setup checks whether the RSS URL already exists in the sheet.

filter column: "W" → your URL/DOI column
Module · LLM Extraction (HTTP)
API key, model, and prompt

Swap the model, revise the schema, and keep the “JSON only” constraint for reliable parsing. Temperature 0 improves consistency.

model: "openai/gpt-4o-mini"
Authorization: "Bearer YOUR_KEY"
Module · Append to Database (Sheets add row)
Column mapping

Map each extracted field into the correct column index. If you add a new column in Sheets, update the mapping object accordingly.

"6": {{12.YourField}}
"21": {{1.url}}
